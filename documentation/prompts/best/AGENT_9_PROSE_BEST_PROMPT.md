This document is the operational prompt contract for Agent 9.
It is intentionally stricter than a narrative guide: it captures what runtime actually sends,
what validators enforce, and what must never drift.

## Scope
- Agent 9 consumes pre-prose intelligence and generates chapter prose JSON.
- Agent 9 is stateful across batches via `NarrativeState`.
- Agent 9 uses deterministic gates and retry feedback loops; prompt quality alone is not enough.

### Required Placeholder Definitions
- `{{BASE_SYSTEM}}`: `MYSTERY_EXPERT_SYSTEM` plus Agent 9 hard additions (cast-name legality, continuity, prose constraints, amateur-detective police naming warning when applicable).
- `{{PROMPT_CONTEXT_BLOCKS_IN_ORDER}}`: concatenation of budget-kept blocks in exact runtime order listed below.
- `{{PROMPT_BUDGET_SUMMARY}}`: runtime token accounting line generated by budgeting stage.
- `{{DEVELOPER_WITH_CONTRACTS}}`: schema contract + word-count contract + novel-quality requirements + quality guardrails + prose requirements.
- `{{ASSET_SELF_REPORT_SUFFIX}}`: append `## Asset Self-Report` instructions only when texture atom IDs are present.
- `{{CHAPTER_OBLIGATION_BLOCK}}`: output of `buildChapterObligationBlock(...)`.
- `{{TIMELINE_STATE_BLOCK}}`: output of `buildTimelineStateBlock(...)`.
- `{{CONTEXT_SUMMARY}}`: output of `buildContextSummary(...)`.
- `{{SANITIZED_SCENES_JSON}}`: `sanitizeScenesCharacters(...)` output, with `estimatedWordCount` clamped to chapter target minimum.
- `{{CHECKLIST_USER_TURN}}`: output checklist user-turn from `buildProsePrompt(...)` including hard-floor words, seasonal allow/forbid words, early-clue checks, chapter-handoff continuity checks (when applicable), pronoun audit table, and JSON-only requirement.
- `{{PREVIOUS_RAW_RESPONSE}}`: previous attempt raw assistant output (only retries before final attempt).
- `{{ENHANCED_RETRY_FEEDBACK}}`: categorized repair directives from `buildEnhancedRetryFeedback(...)`.

## Message Stack Actually Sent
Initial attempt per batch:
1. `system`: `composedSystemWithAssetSelfReport`
2. `user`: `Write the full prose following the outline scenes...` with chapter obligation block, timeline block, context summary, and outline scenes JSON
3. `user`: mandatory pre-output checklist turn (`BEFORE SUBMITTING YOUR JSON`) including hard-floor words, locked facts, season checks, early-clue checks, continuity handoff checks, pronoun audit table, and `Return valid JSON only`

Retry attempts (attempt > 1):
1. same `system`
2. same base `user`
3. same checklist `user`
4. optional `assistant` echo of previous raw response (all retry attempts except the final attempt)
5. retry `user` feedback from `buildEnhancedRetryFeedback(...)`

Important runtime behavior:
- If an attempt fails via thrown error/parse failure, previous assistant raw response is discarded.
- Final attempt avoids assistant echo to force clean rebuild behavior.

## Upstream Intelligence Usage Contract
Agent 9 must use all previously generated intelligence, but not all fields are injected as direct per-chapter raw blocks.

Runtime-equivalent rule:
- Use direct context blocks for structural, logical, and chapter-local constraints.
- Use `world_document` as the creative synthesis layer that carries the substance of upstream outputs.
- Treat all injected material as actionable prose constraints, not optional background.

From `documentation/10_prose_generation/10_prose_generation.md`:
- The guiding ambition is that every pre-prose intelligence item can surface at the right moment and form.
- Some upstream agents are intentionally consumed via synthesis (primarily World Document) rather than direct raw-field injection.
- Asset Library chapter-scoped atomization is the planned mechanism for complete per-atom deployment tracking; not all of that is live yet.

## System Prompt Composition (Runtime)
`system` is assembled as:
`baseSystem + token-budgeted context blocks + PROMPT BUDGET SUMMARY + developerWithContracts`

Where:

- total prompt budget target for assembled system context is `6200` tokens in `applyPromptBudgeting(...)`.
- context blocks are added by priority and can be truncated/dropped by budget policy.
- if texture atoms were selected, runtime appends required end-tag self-report:
  `<used_assets>...</used_assets>`.

## Developer Contract (Runtime-Equivalent)
The `developer` instruction payload is large and dynamic. The mandatory core is:

```text
Return JSON with structure:
{
  "status": "draft",
  "tone": "classic|modern|atmospheric",
  "chapters": [{ "title": "...", "summary": "...", "paragraphs": ["..."] }],
  "cast": ["..."],
  "note": "",
  "audit": {
    "locked_fact_phrases": "...",
    "season_words_used": "...",
    "discriminating_test_present": "...",
    "required_clues_present": "...",
    "early_observation_present": "...",
    "early_inference_present": "..."
  }
}

Requirements include:
- exactly one chapter per outline scene in batch
- chapter numbering starts at chapterStart
- chapter title format: "Chapter N: [Descriptive title]"
- hard scene grounding in first two paragraphs (location + sensory + atmosphere)
- no new facts beyond CML and derived constraints
- word-count contract and obligation contract are non-negotiable
```

Hard behavior constraints included in runtime developer block:
- strict cast-name authenticity (no invented titled names)
- pronoun continuity contract
- chapter-to-chapter continuity handoff (no disconnected reset openings)
- locked fact phrase verbatim requirement
- fair-play clue sequencing constraints
- period authenticity and physical plausibility constraints
- prose requirements from CML injected as hard contract

## Chapter Obligation Contract (Must Pass)
`buildChapterObligationBlock(...)` injects chapter-local obligations including:
- target word count and hard floor framing
- location anchor requirement
- clue obligations for `scene.cluesRevealed`
- early-clue placement obligations for `placement: early` (first 25% + immediate inference beat)
- tonal contrast requirement (except discriminating-test chapter behavior)
- suspect-clearance obligations when configured in prose requirements
- discriminating-test dramatization rule when chapter matches test scene
- locked fact phrase obligations (verbatim policy)
- season lock vocabulary allow/forbid constraints

Note:
- Sensory variant is currently injected as guidance/hinting context, not yet promoted to a hard obligation atom.

## World Document Injection Rules
`buildWorldBriefBlock(...)` enforces chapter-relative injections:
- always: historical moment, story theme, dominant register, chapter arc turning-point register
- humour placement map: scene-position-mapped entry for current arc position
- `breakMoment`:
  - currently injected in the 60% to 85% relative-position window
  - pre-break approach signal (40% to 60%) is planned, not yet implemented
- `revealImplications`:
  - currently injected only at >= 75%
  - early sanitized seeding from chapter 3 is planned, not yet implemented

Known world-brief gap from spec:
- `storyEmotionalArc.arcDescription` is generated upstream but not yet injected in runtime.
- `humourPlacementMap` lookup has an enum mismatch with derived arc-position labels, so humour placement guidance is partial in current runtime.

## Runtime Validation and Retry Gates
All batches are commit-on-success-only.

Deterministic gates (implemented):
1. parse + extraction + chapter count checks
2. chapter structure checks
3. `ChapterValidator.validateChapter(...)` semantic issues
4. `validateChapterPreCommitObligations(...)`
5. pronoun repair + targeted revalidation
6. anti-template linter (`lintBatchProse`) including opening entropy, paragraph fingerprint, n-gram overlap

Underflow path:
- if below hard floor (or configured preferred miss), `attemptUnderflowExpansion(...)` may rewrite chapter
- expansion is revalidated through same gates

Retry path:
- `buildEnhancedRetryFeedback(...)` emits categorized corrective directives
- retries continue until batch passes or attempts exhausted

## NarrativeState Contract
Agent 9 uses a deep-copied `liveNarrativeState` and updates on committed batches only.

Live behavior:
- `continuityTail` is updated in-process after committed batches.
- `lockedFacts` and `characterPronouns` are read as invariants.

Important current limitation:
- `cluesRevealedToReader` is updated on the orchestrator copy, but not fed back into the in-process `liveNarrativeState` during a single `generateProse()` run, so the NSD "already revealed" line remains effectively unfed in that pass.

## Quality Bar (Extra-Clear Operational Standard)
- Every upstream intelligence source must be represented either as direct prompt context or through explicit World Document synthesis; no silent omission of known required constraints.
- If a requirement is listed in chapter obligations, it is a hard generation constraint, not style guidance.
- If a clue is required, prose must surface clue language in scene text, not only implication.
- If a locked fact is referenced, exact phrase fidelity is mandatory.
- If pronoun table and prose disagree, prose is wrong.
- If season lock and scene wording disagree, prose is wrong.
- If retries keep failing, repair directives must be explicit, local, and testable.

## Silent Pre-Output Checklist
- JSON is parseable and shape-complete.
- Exactly one chapter per input scene in the batch.
- Each chapter title matches `Chapter N: ...` format.
- Opening two paragraphs satisfy scene grounding and atmosphere markers.
- Required clues and placement obligations are present.
- Locked fact references are verbatim.
- Pronouns match cast canonical map.
- No invented named entities outside cast.
- No off-season vocabulary against season lock.
