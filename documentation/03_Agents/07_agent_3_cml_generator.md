# Agent 3 - CML Generator

## 1) Role in the Pipeline
Agent 3 generates the primary CML case artifact and coordinates revision/novelty subsystems.

Worker runner:
- `apps/worker/src/jobs/agents/agent3-run.ts`

Prompt/generation implementation:
- `packages/prompts-llm/src/agent3-cml.ts`

## 2) Prompt Design
### 2.1 Prompt Inputs
Key inputs include:
- setting fields (`decade`, `location`, `institution`, `tone`, `weather`, `socialStructure`)
- cast fields (including `castGenders`)
- hard logic directives and devices
- theme and novelty constraints
- `runId`, `projectId`

### 2.2 Prompt Outputs
Primary output:
- full `CaseData` CML object

## 3) Output Contract
Schema:
- `schema/cml_2_0.schema.yaml`

## 4) Scoring and Pass Criteria
- Agent 3 applies internal quality handling in runner flow.
- No standalone scoring adapter in `apps/worker/src/jobs/scoring-adapters` for Agent 3.

## 5) Retry Behavior and Prompt Differences
- Uses embedded Agent 4 revision loop (`reviseCml`) when validation fails.
- Tracks revision attempts and can reduce quality score when revision was required.
- Runs novelty audit stage (Agent 8) with retry semantics in the orchestration flow.

## 6) Downstream Consumers
- Agent 5: `apps/worker/src/jobs/agents/agent5-run.ts`
- Agent 6: `apps/worker/src/jobs/agents/agent6-run.ts`
- Agent 7: `apps/worker/src/jobs/agents/agent7-run.ts`
- Agent 9: `apps/worker/src/jobs/agents/agent9-run.ts`

## 8) Full Field Glossary (Returned Artifact)
Top-level returned fields (`CMLGenerationResult`):

| Field | Type | Required | Meaning |
|---|---|---|---|
| `cml` | `object` | yes | Full CML payload generated by Agent 3 and validated before worker acceptance. |
| `validation` | `object` | yes | Validation outcome metadata for generated CML. |
| `attempt` | `number` | yes | Attempt index of the final successful (or terminal) generation call. |
| `latencyMs` | `number` | yes | End-to-end generation latency in milliseconds. |
| `cost` | `number` | yes | Agent-attributed generation cost. |
| `revisedByAgent4` | `boolean` | no | Indicates whether Agent 4 auto-revision was invoked and applied. |
| `revisionDetails` | `object` | no | Revision metadata including number of revisions and applied changes. |

`validation` fields:

| Field | Type | Required | Meaning |
|---|---|---|---|
| `valid` | `boolean` | yes | Whether generated CML passed schema validation. |
| `errors` | `string[]` | yes | Validation error list used for retry/revision decisions. |

Top-level `cml` fields (`schema/cml_2_0.schema.yaml`):

| Field | Type | Required | Meaning |
|---|---|---|---|
| `CML_VERSION` | `2.0` | yes | Schema version marker for CML payload compatibility. |
| `CASE` | `object` | yes | Core mystery structure used by clue generation, auditing, outlining, and prose generation. |
| `SETTING_REFINEMENT` | `object` | no | Agent 1 enrichment payload when present in full composite CML. |
| `CHARACTER_PROFILES` | `object` | no | Agent 2b enrichment payload when present in full composite CML. |
| `LOCATION_PROFILES` | `object` | no | Agent 2c enrichment payload when present in full composite CML. |
| `TEMPORAL_CONTEXT` | `object` | no | Agent 2d enrichment payload when present in full composite CML. |
| `WORLD_DOCUMENT` | `object` | no | Agent 6.5 world synthesis payload when present in full composite CML. |
| `NARRATIVE_OUTLINE` | `object` | no | Agent 7 outline payload when present in full composite CML. |

`CASE` fields:

| Field | Type | Required | Meaning |
|---|---|---|---|
| `meta` | `object` | yes | Case metadata: title, era, setting, crime class, optional theme. |
| `cast` | `object[]` | yes | Character roster with motives, opportunities, and culpability signals. |
| `culpability` | `object` | yes | Culprit cardinality and final culprit identity list. |
| `surface_model` | `object` | yes | What appears true in-world before hidden mechanism is uncovered. |
| `hidden_model` | `object` | yes | Actual mechanism and outcome behind the apparent facts. |
| `false_assumption` | `object` | yes | Core mistaken assumption driving the puzzle's misdirection. |
| `constraint_space` | `object` | yes | Time/access/physical/social constraints that make inference testable. |
| `inference_path` | `object` | yes | Stepwise observation->correction->effect reasoning chain. |
| `discriminating_test` | `object` | yes | Final test/trap design that distinguishes true from false hypothesis. |
| `fair_play` | `object` | yes | Fair-play guarantees and explanation for reader-solvability. |
| `quality_controls` | `object` | yes | Quantitative requirements for inference, clue timing, and test placement. |
| `prose_requirements` | `object` | no | Optional explicit prose constraints for Agent 9 scene placement and clue mapping. |
