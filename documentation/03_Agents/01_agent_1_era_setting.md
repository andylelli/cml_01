# Agent 1 - Era and Setting Refiner

## 1) Role in the Pipeline
Agent 1 is the first grounding phase. It converts the user brief into a structured historical-constraint artifact that downstream agents can rely on.

Primary implementation:
- `packages/prompts-llm/src/agent1-setting.ts`

Worker runner:
- `apps/worker/src/jobs/agents/agent1-run.ts`

## 2) Prompt Design
### 2.1 Prompt Inputs
Inputs injected into prompt construction:
- `runId`
- `projectId`
- `decade`
- `location`
- `institution` (optional)
- `weather` (optional)
- `socialStructure` (optional)
- `tone` (optional)

Constraint sources used to build prompt content:
- `buildEraConstraints(decade)` from `packages/prompts-llm/src/shared/constraints.ts`
- `buildLocationConstraints(location, institution)` from `packages/prompts-llm/src/shared/constraints.ts`

### 2.2 Prompt Structure
`buildSettingPrompt()` emits three text blocks:
- `system`: role + non-negotiable rules (historical plausibility, no anachronisms, JSON only)
- `developer`: mission, variation directives, explicit schema, quality bar, micro-exemplar, final checklist
- `user`: concrete brief (era/location/tone/etc.) and output instruction

The runtime message payload is two-message chat format:
- system message = `system + developer`
- user message = `user`

### 2.3 What Goes Into the Prompt (Semantically)
- Historical constraint bundle for the requested decade.
- Location/institution constraints relevant to access and investigation feasibility.
- Per-run variation seed directives derived from `runId/projectId`:
  - architectural emphasis
  - naming style
  - focus area
- Full output schema fields the model must satisfy.
- Hard quality rules, including mandatory empty realism error arrays and 3-5 actionable recommendations.

## 4) Scoring and Pass Criteria
Scoring path is enabled through `executeAgentWithRetry()` in:
- `apps/worker/src/jobs/agents/shared.ts`

Agent 1 scoring adapter:
- `apps/worker/src/jobs/scoring-adapters/agent1-scoring-adapter.ts`

Agent 1 scorer:
- `packages/story-validation/src/scoring/phase-scorers/agent1-setting-refinement-scorer.ts`

Scoring dimensions and weights:
- validation: 40%
- quality: 30%
- completeness: 20%
- consistency: 10%

Scorer-level pass gate:
- no critical test failures
- total score >= 60

Global threshold gate (score aggregator):
- default threshold for `agent1-setting-refinement` is 75 in `packages/story-validation/src/scoring/thresholds.ts`
- component minimums also apply:
  - validation >= 60
  - quality >= 50
  - completeness >= 60
  - consistency >= 50

## 5) Retry Behavior and Prompt Differences
Agent 1 has three retry layers with different behavior.

### 5.1 Layer A: Internal generation retries (inside `refineSetting`)
Trigger:
- JSON parse failure
- missing required sections
- non-empty `realism.anachronisms` or `realism.implausibilities`

Prompt delta vs normal attempt:
- none; same base prompt is resent

### 5.2 Layer B: Scoring-driven retries (worker `executeAgentWithRetry`)
Trigger:
- phase score fails threshold/component minimums

Prompt delta vs normal attempt:
- scoring feedback text is generated by `buildRetryFeedback()` in `packages/story-validation/src/scoring/retry-feedback.ts`
- in Agent 1, that feedback is appended as `Retry guidance:` to the `tone` input via `appendRetryFeedbackOptional()` in `apps/worker/src/jobs/agents/agent1-run.ts`

Important nuance:
- this delta is conditional on `tone` existing; if `tone` is undefined, retry guidance is not injected into the Agent 1 prompt input.

### 5.3 Layer C: Schema-repair retry (worker post-check)
Trigger:
- `validateArtifact("setting_refinement", ctx.setting.setting)` fails in `apps/worker/src/jobs/agents/agent1-run.ts`

Prompt delta vs normal attempt:
- none explicit; agent is rerun (`maxAttempts=2`) with same business inputs

## 7) Runtime Traceability
- Hard fail if realism lists remain non-empty after retries.
- Hard fail on schema errors after schema-repair retry.
- Schema warnings are logged to warnings list but do not hard-fail by themselves.

## 9) Produced Files and Prompt Artifacts
This glossary describes the returned `SettingRefinementResult` payload from Agent 1.

Top-level fields:

| Field | Type | Required | Meaning |
|---|---|---|---|
| `setting` | `SettingRefinement` | yes | Structured historical constraint artifact used by downstream agents. |
| `rawResponse` | `string` | yes | Raw model response text captured before typed consumption. |
| `attempt` | `number` | yes | Generation attempt count used for the successful return. |
| `latencyMs` | `number` | yes | End-to-end generation latency in milliseconds. |
| `cost` | `number` | yes | Agent-attributed generation cost from cost tracker. |

`setting` fields:

| Field | Type | Required | Meaning |
|---|---|---|---|
| `era` | `object` | yes | Period constraints for investigation plausibility. |
| `location` | `object` | yes | Physical and access constraints for the setting. |
| `atmosphere` | `object` | yes | Scene tone and sensory baseline metadata. |
| `realism` | `object` | yes | Plausibility checks and correction recommendations. |

`setting.era` fields:

| Field | Type | Required | Meaning |
|---|---|---|---|
| `decade` | `string` | yes | Canonical decade label grounding all constraint lists. |
| `technology` | `string[]` | yes | Period-accurate technology available in-world. |
| `forensics` | `string[]` | yes | Forensic capabilities and limitations for the period. |
| `transportation` | `string[]` | yes | Travel modes and movement constraints. |
| `communication` | `string[]` | yes | Communication channels and delay characteristics. |
| `socialNorms` | `string[]` | yes | Social behavior constraints relevant to interactions and suspicion. |
| `policing` | `string[]` | yes | Law-enforcement practices and procedural limits. |

`setting.location` fields:

| Field | Type | Required | Meaning |
|---|---|---|---|

| Field | Type | Required | Meaning |

| Field | Type | Required | Meaning |
|---|---|---|---|
| `anachronisms` | `string[]` | yes | Detected era mismatches; expected to be empty at success. |
| `implausibilities` | `string[]` | yes | Detected plausibility violations; expected to be empty at success. |
| `recommendations` | `string[]` | yes | Actionable guidance to keep downstream outputs period-credible. |
